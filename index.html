RL 微调让 VLM 分数更好看，也可能让推理链更不可信。

这不是一个单纯的多模态论文条目，而是对“把 VLM 放进机器人闭环”这件事的一次冷水测试。

Original-source digest · 620+ 字

Apple Machine Learning Research 这篇 2026 年 7 月的 ICML 论文讨论的是一个很具体但很关键的问题：当强化学习微调被用于视觉语言模型（VLM）之后，模型在视觉推理 benchmark 上的准确率确实可能提升，但这种提升并不等同于推理过程更可靠。原文的核心实验不是单纯比较分数，而是给模型施加受控文本扰动：一种是误导性标题或说明，另一种是不正确的 chain-of-thought 轨迹。研究发现，经过 RL 调优的 VLM 在面对这些看似轻微的文本干扰时，鲁棒性和置信度会明显下降，开源多模态推理模型的退化尤其突出；闭源模型也表现出类似失败模式，只是整体鲁棒性和推理一致性更强。论文进一步把问题拆到训练动态层面：RL 微调提高 benchmark accuracy 的同时，可能侵蚀 CoT 的忠实性，也就是模型给出的推理过程未必真正支撑最终答案。对抗性增强可以提升一部分鲁棒性，却不能根治忠实性漂移；忠实性感知奖励能部分恢复答案和推理的一致，但和增强手段叠加时又可能让训练坍缩到捷径策略。原文真正的结论不是“RL 不该用于 VLM”，而是仅用准确率评价多模态推理已经不够，训练和评估必须同时看 correctness、robustness 和 visual-grounded reasoning faithfulness。对机器人来说，这个结论很硬：如果 VLM 的语言推理链会被错误上下文牵引，那么它不能未经安全层直接进入动作闭环。

VideoFlexTok 把视频 tokenization 从固定网格推向可变长度记忆。

它离机器人落地还有距离，但它指出了一个长期问题：机器人视觉记忆不能永远按固定帧、固定块、固定 token 成本增长。

展开原文消化与判断

Original-source digest · 650+ 字

VideoFlexTok 的原文从一个基础问题切入：视觉 token 化器不只是压缩像素，它实际上决定了模型保留什么信息、以什么组织方式把信息交给下游模型。传统视频 tokenization 往往把视频切成时空三维网格，每个 token 对应一个局部时空块。这种做法的好处是规则、直接，但代价也很明显：下游模型必须逐块预测大量低层细节，即使这些细节对语义、运动或任务决策并不重要。VideoFlexTok 提出一种可变长度、由粗到细的 token 序列。前几个 token 会涌现式地捕捉语义、运动等抽象信息，后续 token 再逐渐补充纹理、局部细节和视觉精度；生成式 flow decoder 可以用任意数量 token 重建视频。这样一来，系统可以根据任务需求调整 token 数量：如果只需要粗略语义和运动趋势，用少量 token；如果需要高保真重建，再增加 token。原文给出的关键数字是，在类别条件和文生视频任务上，VideoFlexTok 用 1.1B 参数模型达到与 5.2B 模型相当的生成质量，并在 10 秒、81 帧文本到视频训练中只使用 672 个 token，约为可比三维网格 token 化器的八分之一。它的意义不在于马上解决机器人导航，而在于把“视频长度”和“token 成本”部分解耦。对于机器人世界模型、长期视觉记忆、失败回放和场景变化检测，这类表示会逐渐成为底层基础设施。

多智能体团队的问题，不是找不到专家，而是不会使用专家。

这篇研究给“多智能体一定更强”的直觉泼了冷水，也直接对应 command arbitration。

展开原文消化与判断

Original-source digest · 640+ 字

这篇 2026 年 7 月 ICML 论文研究的是自组织多智能体 LLM 团队。它没有预设固定工作流或硬编码聚合规则，而是让多个智能体自由交流，观察团队能否通过互动产生协同效应。原文借用了组织心理学里的问题：团队表现能否达到甚至超过最佳个体成员？实验覆盖受人类启发的基准和前沿机器学习基准，结论相当反直觉：大语言模型团队持续无法匹配专家智能体的单独表现，即使明确告诉团队谁是专家，性能仍会下降；在机器学习基准上，损失最高达到 41.1%。论文进一步分解失败原因，认为瓶颈不在“识别专家”，而在“利用专家”。对话分析显示，团队倾向于整合性妥协：把专家和非专家意见平均化，而不是按专业程度加权。团队规模越大，这种妥协越明显，并且与表现负相关。更微妙的是，这种追求共识的行为会增强对恶意智能体的鲁棒性，说明系统在“对齐”和“利用专业性”之间存在真实权衡。原文真正重要的地方，是把多智能体从数量叙事拉回机制叙事：多个 agent 不是天然带来智能增益，如果没有可靠的 routing、weighting、conflict resolution，团队可能只是把高质量判断稀释成中庸输出。
